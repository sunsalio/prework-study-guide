HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The element refers to what can be found inside the opening and closing tags.
  • The tag refers to whats inside the angled brackets.
  • Attributes control and elements behaviour e.g. src.
  • Semantic HTML elements accurately describe the purpose of the element and what is inside them.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are 3 ways to write CSS in HTML files: Inline, Internal, and external.
  • Inline CSS applies to each element seperately. Internal CSS is placed in head section and styles multiple elements. External CSS placed in seperate file makes changes easier.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch name: creates a new branch and switches to it.
  • git pull origin main: to make sure you're working in the latest version.
  • git push origin branch name: to push changes to remote branch.
  • git add -A: a command that commits all files in the working branch.
  • got commit -m "descriptive message of implemented change": commites the changes.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array is a single variable used to hold a group of data that are related.
  • Strings need to be contained in quotation marks e.g. names.
  • Function do not automatically execute. They must be called by name.
